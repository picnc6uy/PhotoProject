# T-002: Fix .gitignore to actually exclude config.yaml everywhere

## Why
Root `.gitignore` has `/config.yaml` (anchored to root), but the real file lives at `photo_project/src/config.yaml` and was never ignored. This allowed the token leak in T-001. Fix the patterns to catch config.yaml at any depth.

## What
Update `.gitignore` to:
1. Replace `/config.yaml` with `config.yaml` (matches at any level)
2. Add `**/config.local.yaml` (for local overrides)
3. Add explicit `photo_project/src/config.yaml` (defensive)
4. Add `.env`, `.env.*`, `!.env.example` (secrets management)
5. Add `*.log` (noise)
6. Add `dev_data/agent_platform/cache/` (build artifacts)

Verify with `git check-ignore -v photo_project/src/config.yaml`.

## Out of scope
- Cleaning up other ignored files
- Refactoring .gitignore structure beyond what's listed

## Notes
- T-001 already temporarily addressed this; T-002 makes it permanent
- Bug-fix lens: this isn't a logic bug, but a configuration error that had security consequences
