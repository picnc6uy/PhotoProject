# T-006: Fix double-escaped regexes in normalizer / final_exporter

## Why

In raw strings `r"[\\s\\-_/\\.]+"` is a literal backslash + `s`, not the whitespace
character class. Catalog numbers with embedded spaces never get normalized, and
side-suffix stripping in `_base_catalog_number` is similarly broken. The bug
silently corrupts output — there is no error, the rows just do not normalize as
intended. Found in PhotoProject Review, captured as T-006 in FIXES.md.

## What

Two single-character fixes (drop the redundant escape) plus first-time test
coverage for both functions.

**normalizer.py** line 40:

```diff
- text = re.sub(r"[\\s\\-_/\\.]+", "", text)
+ text = re.sub(r"[\s\-_/.]+", "", text)
```

**final_exporter.py** line 72 (`_base_catalog_number`):

```diff
- return re.sub(r"([-\\s])([AB])$", "", catalog.strip(), flags=re.IGNORECASE).strip()
+ return re.sub(r"[-\s]([AB])$", "", catalog.strip(), flags=re.IGNORECASE).strip()
```

Failing test table (per the bug-fix lens, add these tests BEFORE the fix and
commit a red-test commit; then commit the fix; then commit any green-test
follow-up if structure changed):

| input | expected normalized | expected base catalog |
|---|---|---|
| `"K 12 345 -A"` | `"K12345A"` | `"K 12 345"` |
| `"DGG.413 / 122-2"` | `"DGG4131222"` | `"DGG.413 / 122-2"` (no AB suffix to strip) |

Create two new test files:

- `photo_project/tests/test_normalizer.py`
- `photo_project/tests/test_final_exporter.py`

Run `pytest -q` after each commit and capture output for the verification doc.
