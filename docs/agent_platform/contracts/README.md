# Agent Contracts

Each agent should have a concise contract describing purpose, inputs, outputs, and safety considerations. These briefs are meant to be quick references for developers and orchestrator maintainers.

The following contract files are available:

- `task_refiner.md`
- `risk_monitor.md`
- `planner_agent.md`
- `design_advisor.md`
- `implementer_agent.md`
- `test_runner_agent.md`
- `code_reviewer.md`
- `requirements_verifier.md`
- `red_team_reviewer.md`
- `integrator_agent.md`
- `release_coordinator.md`
- `post_merge_observer.md`
- `agent_architect.md`
